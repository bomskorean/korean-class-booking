## 0. 한눈에 보는 요약 (TL;DR)

- **무엇을**: 한국어교실의 온라인 예약·티켓·결제 시스템
- **누가**: 일본인 학습자(학생) + 강사/운영자(관리자)
- **어디서**: 기존 홈페이지에 버튼 링크 + LINE 공식계정에 링크 → **별도 창(새 탭/LIFF)** 으로 예약 페이지 오픈
- **핵심 기능**: 예약(무료체험·정규수업), 잔여 티켓 차감, 코스 소개, 진행상황(수강 이력), 예약 완료/취소/변경 알림(이메일·LINE)
- **티켓**: 4회 / 8회 / 12회 패키지, 수업 1회당 1회 차감
- **결제**: 오프라인(현장/계좌) + 온라인(Stripe 카드, PayPay)
- **코스**: 정규반(한글부터), 단기코스(여행·드라마·팬미팅), TOPIK 준비, 유학준비(대학)

## 1. 목표와 배경

### 1.1 목적
학생이 홈페이지나 LINE에서 링크 한 번으로 수업을 예약하고, 보유 티켓이 자동 차감되며, 예약/취소/변경 시 이메일·LINE으로 안내를 받는 시스템을 만든다. 운영자는 예약 현황과 학생별 진행상황을 한 곳에서 관리한다.

### 1.2 사용자 정의
- **게스트(미가입)**: 코스 소개 열람, 무료체험 예약(이메일/LINE만으로 신청)
- **학생(가입)**: 로그인 후 정규수업 예약, 티켓 잔여 확인, 수강 이력 확인, 예약 변경/취소
- **관리자(강사/운영자)**: 시간표 개설, 예약 승인/관리, 티켓 부여·차감, 결제 확인, 학생 관리, 통계 확인

### 1.3 성공 기준 (KPI)
- 무료체험 신청 → 정규 등록 전환율 측정 가능
- 예약 노쇼(No-show)율 감소(리마인드 알림)
- 운영자의 수기 관리(엑셀·메신저) 시간 50% 이상 절감

## 2. 시스템 구성 개요

```
[홈페이지 bomskankokugo.com] --링크--> ┐
                                       ├─> [예약 웹앱 (별도 창/새 탭)]
[LINE 공식계정 (리치메뉴/링크)] --링크--> ┘        │
                                                   ├─ 예약/티켓/코스/진행상황
                                                   ├─ 결제 (Stripe / PayPay)
                                                   └─ 알림 발송 (Email / LINE Push)
                                                          │
                                          [관리자 대시보드]
```

- 예약 웹앱은 **독립 도메인 또는 서브도메인**으로 운영(예: `reserve.bomskankokugo.com` 또는 `booking.bomskankokugo.com`).
- 홈페이지/LINE의 링크는 `target="_blank"` 로 **새 창**에서 열리도록 한다.
- LINE 내부에서 자연스럽게 열려면 **LIFF(LINE Front-end Framework)** 또는 **LINE MINI App** 으로 구성하면 LINE 앱 안에서 바로 동작하고 LINE 로그인·푸시 연동이 매끄럽다.

#### 진입 링크 분리 (무료체험 ↔ 정규수업)

두 예약을 **완전히 분리된 별도 페이지/링크**로 운영하고, 각 링크는 **독립된 새 창**으로 열린다.

| 구분 | 링크(예) | 용도 | 로그인 |
|---|---|---|---|
| **무료체험** | `https://reserve.bomskankokugo.com/trial` | 비회원 신청 + 인테이크 설문, 정각 슬롯, 티켓 불필요 | 불필요 |
| **정규수업** | `https://reserve.bomskankokugo.com/regular` | 티켓 보유자 예약, 30분 그리드, 티켓 차감 | 필요 |

- **홈페이지**: 버튼 2개를 따로 배치 — "무료体験を予約" → `/trial`, "レッスンを予約" → `/regular`. 둘 다 `target="_blank" rel="noopener"`.
- **LINE**: 리치메뉴 칸을 2개로 나눠 각각 `/trial`, `/regular` 링크. LIFF로 만들 경우 **LIFF 앱을 2개**(trial용·regular용) 등록하면 각각 독립 창으로 열림.
- 두 페이지는 헤더·디자인은 공유하되 플로우·검증·시간 규칙이 분리됨(서로 이동 링크는 두지 않음, 또는 작게만).

## 3. 기능 요구사항

### 3.1 예약 (Booking)

#### 예약 종류
1. **무료체험 (無料体験)**
   - 비회원도 신청 가능(이름·연락처·희망일시·관심코스 입력)
   - 1인 1회 제한(이메일/전화번호 기준 중복 체크)
   - 티켓 차감 없음
   - **신청 시 인테이크 설문 작성**(아래 3.8) → 강사가 첫 수업 전 학생 배경 파악
2. **정규수업 (正規レッスン)**
   - 로그인 필수, 보유 티켓이 1회 이상 있어야 예약 가능
   - 예약 확정 시 티켓 1회 자동 차감, 취소 시 정책에 따라 복원

#### 예약 플로우
1. 코스 선택 → 2. 강사/시간대(슬롯) 선택 → 3. (정규) 티켓 확인 → 4. 예약 확정 → 5. 알림 발송
- 수업 형태: **대면 / 온라인(Zoom·Google Meet 링크 자동첨부) / 둘 다** 선택 옵션
- 정원 관리: 슬롯별 정원(1:1 또는 그룹) 설정, 마감 시 예약 불가 표시

#### 예약 변경/취소 (계약서 제7·8조 기준)
- 학생이 마이페이지에서 직접 변경·취소
- **24시간 전까지**: 일정 변경·취소 무료, **1회에 한해**(티켓 보류 해제). 시스템이 "무료 취소 1회 사용" 여부를 기록.
- **24시간 이내 취소·당일 취소·노쇼(No-show)**: 해당 수업 **1회 차감**(보류 → 차감 확정).
- **지각**: 종료 시각 연장 없음, 그래도 **1회 차감**.
- **반복 당일취소·노쇼**: 시스템이 횟수를 카운트해 일정 횟수 초과 시 **예약 제한 플래그** → 운영자에게 알림(계약 해지·예약 제한 근거).
- 변경 = 기존 슬롯 취소 + 신규 슬롯 예약(티켓 이동, 추가 차감 없음, 24시간 룰 적용).

### 3.2 잔여 티켓 (Ticket)

- **가격·유효기간표 (확정 — 1회 50분, 全て税込)**

  | 상품 | 가격(税込) | 1회당 단가 | 유효기간 |
  |---|---|---|---|
  | 1회 | 4,000 | 4,000 | 2개월 |
  | 2회 | 8,000 | 4,000 | 2개월 |
  | 3회 | 12,000 | 4,000 | 2개월 |
  | 4회권 | 15,600 | 3,900 | **3개월** |
  | 5회권 | 19,250 | 3,850 | **3개월** |
  | 6회권 | 22,800 | 3,800 | **4개월** |
  | 7회권 | 26,250 | 3,750 | **4개월** |
  | 8회권 | 29,600 | 3,700 | **5개월** |
  | **12회권 (신설)** | **42,000** | **3,500** | **7개월** |

  - **할인 패턴**: 4회권부터 1회당 단가가 회당 **-50엔**씩 체감(3,900 → 3,700). 동일 패턴 연장 시 12회는 3,500엔(8회 대비 -200엔) → **12회권 = 12 × 3,500 = 42,000엔**.
  - **유효기간(여유 설계)**: "월 2회 최소 페이스 + 약 1개월 버퍼" 기준으로 넉넉하게 설정. 회수가 많을수록 길어짐.
  - 가격은 **세금 포함 총액(税込)** 표기(일본 総額表示義務). 경과한 미사용분은 실효(법령 허용 범위 외 현금 환불 없음).
  - ※ 12회권 = 42,000엔 < 5만엔이라, 유효기간을 2개월 넘게 늘려도 '특정계속적역무'(語学教室, 기간 2개월 초과 **및** 5만엔 초과 시 적용) 대상에는 해당되지 않음. 다만 향후 5만엔 이상 상품 추가 시 재검토 필요(전문가 확인 권장).
- **티켓 차감 시점 (계약서 제6조)**: 원칙적으로 **수업 종료 시 1회 차감**. 단, 시스템 운영상 예약 시점에 1회를 **보류(hold)** 해 잔여를 확보하고, 수업 완료/노쇼/지각 시 **차감 확정**, 24시간 전 취소 시 **보류 해제(복원)** 로 처리.
- 마이페이지에 **잔여 횟수 + 유효기간(D-day) + 사용 이력** 표시
- 잔여 1~2회 또는 만료 7일 전 **재구매·소진 안내 알림** 자동 발송

> ✅ **전부 확정됨**: 12회권 신설(42,000엔), 유효기간 차등(위 표), 가격 税込, **유효기간 기산점 = 첫 수업일**.
> - **남용 방지 가드(권장)**: 구입일로부터 **30일 내 첫 수업 미실시** 시, 구입일을 기산점으로 간주(만료 무한 연기 방지). 운영자 설정값.
> - 계약서 제5조 문구도 "購入日" → "**初回レッスン実施日**"(+30일 가드)로 수정 제안.

### 3.3 코스 소개 (Course Catalog)

각 코스: 제목, 설명, 대상 레벨, 수업 시간/횟수, 가격(티켓 연계), 대표 이미지, 강사 소개, 신청 버튼.

1. **정규반 (正規クラス) — 한글부터 차근차근**
   - 레벨: 입문 → 초급 → 중급 → 고급 (단계별 커리큘럼)
2. **단기코스 (短期コース)**
   - 여행 한국어 / 드라마로 배우는 한국어 / 팬미팅·K-POP 한국어
   - 회차·기간 한정, 목적 중심 회화
3. **TOPIK 준비**
   - 급수별(초급/중급/고급) 대비, 모의시험·기출 중심
4. **유학준비 (留学準備) — 대학**
   - 대학 진학 목적, 면접·서류·아카데믹 한국어

### 3.4 진행상황 (Progress)

- 학생별 **수강 이력**: 날짜, 코스, 강사, 출결(출석/결석/지각), 사용 티켓
- **레벨/단계 진척도**: 현재 단계, 완료 차시, 다음 목표
- 강사 **피드백·메모**(수업 후 코멘트, 숙제)
- (권장) 간단한 **학습 진도 게이지**와 수료증 발급 기능

### 3.5 알림 (Notifications)

발송 채널: **이메일 + LINE 푸시 메시지** (학생이 선호 채널 선택 가능)

| 이벤트 | 발송 시점 | 내용 |
|---|---|---|
| 예약 완료 | 예약 확정 즉시 | 일시·코스·강사·장소(또는 온라인 링크)·취소 정책 |
| 예약 변경 | 변경 확정 즉시 | 변경 전/후 일시 |
| 예약 취소 | 취소 즉시 | 취소 내역·티켓 복원 여부 |
| 리마인드 | 수업 24h·1h 전 | 곧 시작하는 수업 안내 |
| 결제 완료 | 결제 성공 시 | 영수증·티켓 부여 내역 |
| 티켓 소진/만료 임박 | 잔여 1~2회·만료 7일 전 | 재구매 안내 |
| 무료체험 팔로업 | 체험 후 당일·3일·7일 | 정규 등록 할인 안내(24·25번) |
| 구글 리뷰 요청 | 무료체험 완료 1~3h 뒤 | 리뷰 작성 링크(3.11) |
| 복습자료 추가 | 강사 업로드 시 | 자료 열람 안내(3.10) |
| 월정액 결제·갱신 | 매월 결제일 | 결제 완료·다음 예약 안내(25번) |

- 이메일: 트랜잭션 메일 서비스(예: Resend, SendGrid, Amazon SES) 사용
- LINE: **Messaging API의 Push Message** 사용(학생 LINE userId 보유 필요 → LINE 로그인/공식계정 친구추가로 수집)

### 3.6 결제 (Payment)

- **오프라인**: 현장 현금/카드, 계좌이체 → 관리자가 입금 확인 후 티켓 수동 부여(상태: 입금대기/완료)
- **온라인**:
  - **Stripe**: 신용/체크카드 결제
  - **PayPay**: 2025년 4월부터 Stripe를 통해 PayPay 결제 가능(체크아웃에서 PayPay 선택 → QR 스캔). 따라서 **Stripe 한 곳으로 카드+PayPay 모두 처리** 가능 → 연동 단순화
- 결제 성공 → 티켓 자동 부여 + 결제완료 알림 + 영수증
- 환불 정책 명시(부분 사용 패키지 환불 규정 설정값)

### 3.7 수업 시간·슬롯 규칙 (중요)

다른 플랫폼(italki, Preply, 줌 개인수업 등)에서도 수업하므로 **이중 예약(더블부킹)을 막는 것**이 핵심이다. 모든 가용시간은 **구글 캘린더를 단일 기준(Single Source of Truth)** 으로 삼는다.

#### 시간 정의 (확정)

| 항목 | 학생에게 표시 | 실제 캘린더 점유(블록) | 시작 시각 그리드 | 구성 |
|---|---|---|---|---|
| 무료체험 | **50분** | **60분** | **정각만 (60분 단위)** 예: 10:00, 11:00 … | 수업 50분 + 뒤 10분 제외(쉬는시간) |
| 정규수업 | **50분** | **70분 (1시간 10분)** | **30분 단위** 예: 10:00, 10:30, 11:00 … | 수업 50분 + 여유 버퍼 20분 |

- 학생 화면에는 **항상 50분 수업**으로 보인다.
- 캘린더 점유·다른 플랫폼 차단에는 **실제 블록(60분/70분)** 을 사용한다.
- 설계 의도: 무료체험은 시간 낭비 없이 **정각에 시작**해 깔끔히 1시간 안에 마무리, 정규수업은 **30분 단위로 유연하게** 잡되 블록을 넉넉히 둬 여유 있게 진행.

#### 예약 가능 슬롯 규칙
- **무료체험**: 시작 시각은 **정각(:00)** 에만 노출. 한 건 예약 시 블록 60분 점유.
- **정규수업**: 시작 시각은 **30분 단위(:00, :30)** 로 노출. 한 건 예약 시 블록 70분 점유.
- 예약 확정 시 점유 블록과 겹치는 그리드 슬롯은 자동으로 예약 불가 처리.
- 다음 예약 가능 시작 시각 = **점유 블록이 끝난 뒤 첫 그리드 지점**.

#### 예시 타임라인

**무료체험 (정각 그리드, 블록 60분, 표시 50분)**
```
10:00 예약 → 점유 10:00~11:00 (학생 표시 10:00~10:50)
  → 다음 예약 가능 시작 = 11:00
```

**정규수업 (30분 그리드, 블록 70분, 표시 50분)**
```
10:00 예약 → 점유 10:00~11:10 (학생 표시 10:00~10:50)
  → 10:30, 11:00 슬롯 자동 차단 (블록과 겹침)
  → 다음 예약 가능 시작 = 11:30
```
```
10:30 예약 → 점유 10:30~11:40 (학생 표시 10:30~11:20)
  → 11:00, 11:30 슬롯 자동 차단
  → 다음 예약 가능 시작 = 12:00
```

#### 구글 캘린더 양방향 동기화 (더블부킹 방지)
1. **예약 → 캘린더 쓰기**: 본 시스템에서 예약 확정 시 강사의 구글 캘린더에 **블록 길이(60/70분)** 만큼의 일정(이벤트)을 자동 생성한다.
2. **캘린더 → 예약 막기(읽기)**: 구글 캘린더에 **이미 일정이 있는 시간**(다른 플랫폼 수업, 개인 일정 등)은 본 시스템에서 예약 슬롯으로 노출하지 않는다(Free/Busy 조회).
3. **다른 플랫폼 차단**: 다른 플랫폼이 같은 구글 캘린더를 가용시간 기준으로 참조(2-way sync 지원 플랫폼)하면, 본 시스템 예약 → 캘린더 점유 → 다른 플랫폼에서도 자동 차단된다.
   - ※ 일부 플랫폼은 캘린더 동기화를 지원하지 않거나 일방향만 지원할 수 있으므로, 각 플랫폼의 캘린더 연동 지원 여부를 사전 확인 필요(미지원 시 수동 차단 보완).
4. **충돌 방지 락**: 동시 예약 시도 시 캘린더 쓰기 전에 슬롯 점유 락(DB 트랜잭션 + Free/Busy 재확인)으로 중복 확정을 막는다.

#### 운영자 설정값(권장 기본값)
- 영업시간(예: 평일 10:00~21:00 등) 내에서만 30분 그리드 생성
- 무료체험 휴식: 10분 / 정규 버퍼: 20분 (변경 가능)
- 하루 최대 수업 수, 연속 예약 사이 최소 공백 등 옵션화

### 3.8 무료체험 신청 설문 (인테이크)

무료체험 예약 시 함께 받는 설문. 강사가 첫 수업 전에 학생의 목표·수준·배경을 파악해 맞춤 안내하고, 정규 등록 시 추천 코스 매칭에도 활용한다. 입력은 일본어 기본(필요 시 한국어 병기).

**필수 항목**
1. **학습 희망 기간 (学習希望期間)** — 단일 선택
   - 1개월 / 3개월 / 6개월 / 1년 이상 / 미정
2. **학습 동기 (学習のきっかけ)** — **복수 선택 가능**
   - K-POP / 유학(留学) / 여행(旅行) / 드라마(ドラマ) / 취미(趣味)
   - (추가 권장: 팬미팅·콘서트, 비즈니스·취업, 한국 거주·이주, 한국 가족·지인)
3. **학습 경험 (学習経験)** — 단일 선택
   - 완전 입문(처음) / **독학** / **한국어교실(학원) 수강 경험** / 독학+학원 모두
4. **사용 교재 (使用教材)** — 위에서 독학·학원 선택 시 표시(조건부 필드)
   - 교재명 자유 입력 + 자주 쓰는 교재 선택지(예: 「できる韓国語」, 「hime式」, 세종한국어, 연세/서강/이대 한국어, TOPIK 문제집 등) + "기억 안 남" 옵션

**선택 항목(권장)**
5. **자가 진단 레벨** — 입문 / 초급 / 중급 / 고급, 또는 간단 자가 체크(한글 읽기 가능? 자기소개 가능? 등)
6. **희망 수업 형태** — 대면 / 온라인 / 상관없음
7. **목표·하고 싶은 말 (자유 입력)** — 예: "드라마를 자막 없이 보고 싶다"
8. **알게 된 경로** — 홈페이지 / LINE / SNS / 지인 소개 (마케팅 분석용)

**활용**
- 응답은 학생 프로필에 저장 → 강사 대시보드에서 첫 수업 전 열람
- 동기·경험 데이터로 **추천 코스 자동 매칭**(예: 동기=유학 → 유학준비반, 동기=드라마/취미 → 단기코스)
- 통계: 동기·유입경로별 전환율 분석

### 3.9 계약·약관 및 정책 (受講契約書 반영)

업로드된 受講契約書(회수권 계약서)의 조항을 시스템 기능으로 반영한다.

- **전자 동의/계약 (제14조)**: 티켓 첫 구매(또는 회원가입) 시 계약서 내용을 표시하고 **체크 동의 + 타임스탬프 기록**(전자적 동의). 동의 이력 저장(버전·일시·IP). 필요 시 PDF 계약서 자동 생성·메일 발송.
- **사전 결제 원칙 (제4·6조)**: 입금/결제 확인 후에만 예약 가능. 오프라인 입금은 "입금대기 → 운영자 확인 → 티켓 활성화" 플로우.
- **티켓 양도·공유 금지 (제9조)**: 티켓은 본인만 사용. 시스템상 계정 간 양도 기능 없음(운영자 수동 이전만 로그 기록).
- **중도 해약·환불 (제10조)**: 운영자가 환불 처리 시 **기제공분을 정가(1회 4,000엔)로 정산** → 잔액 환불 자동 계산. 해약 수수료·실비(이체 수수료 등) 차감 입력란. 교재비·특별프로그램비는 환불 대상 외 처리.
- **휴강·대체수업 (제11조)**: 천재지변·강사 급병·플랫폼 장애 등으로 휴강 시, 영향받은 예약 **일괄 취소 + 티켓 자동 복원** 또는 **유효기간 연장**, 학생에게 일괄 알림.
- **성과 비보증·면책 (제12조)**: 약관 문구에 명시(시스템 고지 텍스트).
- **관할·분쟁 (제13조)**: 약관 푸터에 표기.

### 3.10 복습자료 라이브러리

학생이 지난 수업의 자료를 다시 볼 수 있는 개인 라이브러리.

- **자료 유형**: 수업 녹화 영상(온라인 수업), 교재·PDF, 숙제, 강사 메모, 단어장.
- **연결**: 각 자료는 학생(또는 예약/수업)과 연결. 강사가 수업 후 업로드 → 해당 학생 마이페이지에 노출.
- **저장**: 파일은 Google Drive 또는 클라우드 스토리지(S3 등)에 저장하고 링크/권한만 관리(대용량 영상 대비). DB에는 메타데이터(제목·유형·URL·공개여부)만.
- **접근 제어**: 본인 자료만 열람. 유효기간 만료·퇴원 후 접근 정책은 설정값.
- **알림**: 자료 업로드 시 "復習資料が追加されました" 안내(LINE/메일).
- 화면: 마이페이지 내 "復習資料" 탭(수업 날짜별 정렬, 코스 필터).

### 3.11 무료체험 후 Google 리뷰 유도

- **시점**: 무료체험 수업이 **完了(completed)** 상태가 된 직후(예: 종료 1~3시간 뒤) 자동 메시지.
- **채널**: LINE 푸시 + 메일. "本日はありがとうございました！よろしければGoogleでレビューをお願いします🙏" + **Google 리뷰 작성 링크**.
- **링크**: Google 비즈니스 프로필의 리뷰 작성 단축 링크(`https://g.page/r/...../review` 형식)를 사용.
- **남용 방지**: 1인 1회만 발송, 이미 보낸 학생 제외. 강요 아닌 권유 톤.
- **연계**: 25번(정규 전환 팔로업)과 분리 발송하거나 묶어서 발송. 리뷰 작성 유도와 첫 등록 할인 안내를 함께.
- **관리**: 발송/클릭 여부를 통계로(리뷰 요청 → 작성 전환은 Google 정책상 직접 추적 불가, 클릭까지만 측정).

## 6. 연동 가이드 (Claude Code 구현 참고)

### 6.1 LINE 연동
- **LINE 공식계정(Official Account)** 생성 → **Messaging API 채널** 발급
- **리치메뉴**에 "예약하기" 버튼 → 예약 웹앱 URL 연결
- 예약 페이지를 **LIFF 앱**으로 등록하면 LINE 앱 내에서 열리고, **LINE 로그인**으로 `userId` 확보 → Push Message 발송 가능
- 친구추가 시 `userId` 저장 → 알림 발송 키로 사용
- 단순 링크만 원하면: 공식계정 메시지/리치메뉴에 `https://reserve.bomskankokugo.com?from=line` 형태로 새 창 오픈

### 6.2 결제 연동 (Stripe)
- Stripe 계정(일본) 개설 → Payment Methods에서 **카드 + PayPay** 활성화
- **Stripe Checkout** 또는 **Payment Intents** 사용
- Webhook(`payment_intent.succeeded`)로 결제 확인 → 티켓 자동 부여 + 알림 발송
- PayPay는 Stripe 체크아웃 화면에서 선택 → QR 스캔 결제(별도 SDK 불필요)

### 6.3 이메일 연동
- 트랜잭션 메일(Resend/SendGrid/SES) — 템플릿(일본어 기본, 한국어 옵션)

### 6.4 구글 캘린더 동기화 (필수 — 더블부킹 방지)
- **Google Calendar API** 사용. 강사 계정 OAuth 연동 후 전용 캘린더 1개를 "수업 가용시간 기준"으로 지정.
- **읽기(Free/Busy)**: 슬롯 생성 시 해당 시간대에 기존 일정이 있으면 예약 불가로 처리 → 다른 플랫폼 수업/개인 일정 자동 회피.
- **쓰기**: 예약 확정 시 블록 길이(60/70분) 이벤트 생성, `gcal_event_id` 저장. 취소 시 이벤트 삭제, 변경 시 이벤트 이동.
- **동시성**: 캘린더 쓰기 직전 Free/Busy 재확인 + DB 락으로 중복 확정 방지.
- **웹훅(권장)**: Google Calendar push notification으로 외부 일정 변경을 실시간 반영.
- ※ 상세 시간 규칙은 **3.7 수업 시간·슬롯 규칙** 참조.

## 7. 권장 기술 스택 (제안 — Claude Code 친화적)

- **프론트엔드**: Next.js (React) + TypeScript + Tailwind CSS
- **백엔드/DB**: Next.js API Routes 또는 별도 Node 서버 + PostgreSQL (Prisma ORM)
- **인증**: NextAuth (이메일 + LINE Login Provider)
- **결제**: Stripe SDK (카드 + PayPay)
- **알림**: Resend(이메일) + LINE Messaging API SDK(`@line/bot-sdk`)
- **호스팅**: Vercel(웹) + Supabase/Neon(DB) — 소규모 운영에 비용 효율적
- **i18n**: `next-intl` 등으로 일본어 기본 + 향후 한국어 추가 용이

> 스택은 제안이며, 운영자가 PHP/워드프레스 기반을 선호하면 예약 플러그인(예: Amelia, Bookly) + LINE/Stripe 연동으로도 구현 가능. 단, 티켓 차감·진행상황 등 커스텀 로직은 코드 기반(Next.js)이 유연합니다.

## 8. 단계별 개발 순서 (Roadmap)

**MVP (1차)** — 핵심 예약 흐름
1. 코스 소개 페이지 + 무료체험 신청(비회원)
2. 회원가입/로그인(이메일·LINE)
3. 슬롯/예약(정규) + 티켓 차감
4. 마이페이지(잔여 티켓·예약 목록)
5. 이메일 알림(완료/취소/변경) + 관리자 기본 화면

**2차** — 결제·알림 고도화
6. Stripe 카드 + PayPay 온라인 결제 → 티켓 자동 부여
7. LINE Push 알림 + 리마인드(24h·1h 전)
8. 관리자 시간표/결제/학생 관리 대시보드

**3차** — 운영 최적화
9. 진행상황·피드백·수료증
10. 통계 대시보드, Google Calendar 동기화
11. 재구매 유도·쿠폰·추천인 등 마케팅 기능

## 9. 채택 기능 목록 (전부 반영)

아래 항목은 **모두 개발 범위에 포함**한다(요청: "추가하면 좋을 기능 전부 반영"). 우선순위는 8. 로드맵 참고.

운영·전환·편의를 높이는 항목들:

1. **리마인드 알림 + 노쇼 관리**: 수업 24h/1h 전 자동 안내로 노쇼 감소. 노쇼 시 티켓 차감 정책.
2. **대기자 명단(Waitlist)**: 마감된 인기 슬롯에 빈자리 생기면 자동 알림.
3. **쿠폰·할인코드**: 무료체험 후 첫 등록 할인, 친구추천(레퍼럴) 할인.
4. **다국어(일↔한) 토글**: 일본어 기본, 한국어 학습자/강사용 전환.
5. **레벨 테스트**: 가입 시 간단 진단 → 추천 코스 자동 매칭.
6. **그룹/1:1 구분**: 슬롯 정원과 가격을 형태별로 분리.
7. **온라인 수업 자동 링크**: 예약 확정 시 Zoom/Meet 링크 자동 생성·첨부.
8. **수료증·뱃지**: 단계 완료 시 PDF 수료증 발급으로 동기부여.
9. **리뷰·후기**: 수업 후 별점/후기 → 코스 페이지 노출(마케팅).
10. **관리자 모바일 알림**: 신규 예약·입금 시 운영자에게 LINE/메일 통지.
11. **출석 QR 체크인**: 대면 수업 현장 QR로 출석 자동 기록.
12. **개인정보·약관 동의 & GDPR/일본 개인정보보호법 대응**: 가입·결제 시 동의 절차.
13. **환불·휴강 정책 자동화**: 강사 사정 휴강 시 티켓 자동 복원·일괄 알림.
14. **간단한 매출/전환 리포트**: 월별 신규·재구매·매출 자동 집계.

**계약서 반영 항목 (필수)**
15. **전자 계약 동의**: 첫 구매 시 受講契約書 표시·체크 동의·타임스탬프·PDF 발행(제14조).
16. **중도해약 환불 계산기**: 기제공분 정가(4,000엔) 정산 + 수수료 차감 자동 계산(제10조).
17. **휴강 일괄 처리**: 강사 휴강 시 영향 예약 일괄 취소·티켓 복원·유효기간 연장·일괄 알림(제11조).
18. **양도 금지·반복 노쇼 관리**: 티켓 본인 한정, 노쇼·당일취소 카운트 → 예약 제한(제7·9조).

**추가 신규 제안 (전부 채택)**
19. **다회차 묶음 예약**: 티켓 보유분을 한 번에 여러 회차(예: 매주 같은 요일·시간) 일괄 예약.
20. **세금계산서/영수증(일본 인보이스 제도, 適格請求書) 대응**: 결제 영수증·인보이스 번호 발행.
21. **학생 친구추가 유도(LINE)**: 예약 완료 화면에서 공식계정 친구추가 QR·버튼으로 푸시 수신 동의 수집.
22. **강사용 일일 브리핑**: 매일 아침 오늘 수업·신규 무료체험·입금대기 요약을 LINE/메일로 자동 발송(스케줄).
23. **언어 표기**: 화면·알림·계약서 일본어 기본, 한국어 병기 옵션.

**전환·리텐션 (추가 채택)**
24. **무료체험 → 정규 자동 팔로업**: 체험 완료 후 당일·3일 뒤 자동 메시지(LINE/메일) + 첫 등록 한정 할인 쿠폰. 미전환 시 7일 뒤 리마인드.
25. **월정액(月謝制) 옵션**: 회수권 외에 "매주 고정 요일·시간 1회 = 월정액" 구독형 상품. Stripe Subscription으로 매월 자동결제, 고정 슬롯 자동 예약(스킵/일시정지 가능).
26. **포인트·스탬프 적립**: 수강 1회마다 스탬프, N개 적립 시 무료 1회 또는 할인. 재구매 동기 부여.
27. **휴면 학생 리engagement**: 유효기간 만료·일정기간 미예약 학생에게 자동 재방문 쿠폰·안내.

**운영 효율 (추가 채택)**
28. **강사 다중 스케줄 관리**: 강사별 가용시간·전용 구글 캘린더 분리. 슬롯·예약을 강사 단위로 관리(확장 대비).
29. **학생용 캘린더 추가(.ics)**: 예약 확정 시 학생이 본인 Google/Apple 캘린더에 원클릭 등록 → 노쇼 감소.
30. **LINE 자동응답 FAQ 봇**: 영업시간·가격·접근(위치)·결제방법 등 자주 묻는 질문 자동 응답으로 문의 부담↓.

**해외·온라인 (추가 채택)**
31. **타임존 표시·변환**: 한국·해외 거주 온라인 학생에게 현지시각 자동 변환 표시. 예약 착오 방지.
32. **복습자료 라이브러리** (3.10 상세): 수업 녹화·교재·숙제를 학생별로 공유.

**리뷰·평판 (추가 채택)**
33. **무료체험 후 Google 리뷰 유도** (3.11 상세): 체험 완료 직후 Google 리뷰 작성 링크 안내로 평판·검색 유입 강화.

## 10. 비고 / 의사결정 필요 항목

Claude Code 개발 전에 운영자가 확정하면 좋은 값들:
- ✅ 가격(税込)·12회권(42,000엔)·유효기간(차등)·기산점(첫 수업일, 30일 가드) 모두 확정.
- 취소·노쇼 정책은 계약서(24시간·1회 무료·노쇼 차감)대로 반영 — 추가 변경 시만 결정
- 영업시간(예약 가능 시간대)과 그룹 정원
- 가용시간 기준이 될 **구글 캘린더 계정/전용 캘린더** 지정
- 예약 웹앱 **도메인/서브도메인** 결정
- LINE 공식계정 보유 여부(없으면 신규 개설 필요)
- Stripe 일본 계정 개설 주체(개인/법인)
